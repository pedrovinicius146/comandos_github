Bem-vindo ao Exemplo de Projeto Git

Sobre este projeto

Este projeto foi criado para demonstrar o uso básico do Git no controle de versão de arquivos de um projeto web.
Aqui você pode modificar o conteúdo deste arquivo e versioná-lo utilizando Git.

O que você pode fazer?

  • Criar novas seções nesta página.
  • Modificar o texto ou o estilo da página.
  • Fazer commit das alterações e enviá-las para um repositório remoto.
  • Criar uma nova branch.

Comandos Git

Comando Git Funcionalidade
git init Inicializa um novo repositório Git em um diretório.
git clone <url> Clona um repositório remoto para o local.
git status Mostra o estado atual do repositório, incluindo arquivos modificados, adicionados ou não rastreados.
git add <arquivo> Adiciona um arquivo específico para ser preparado (staged) para o próximo commit.
git add . Adiciona todos os arquivos modificados no diretório atual para a área de preparação (staging area).
git commit -m "<mensagem>" Confirma as mudanças preparadas com uma mensagem descritiva.
git commit --amend Altera o último commit, permitindo a correção de mensagens ou inclusão de arquivos esquecidos.
git log Exibe o histórico de commits no repositório.
git diff Mostra as diferenças entre arquivos modificados mas não preparados ou entre commits.
git branch Lista as ramificações (branches) do repositório.
git branch <nome> Cria uma nova ramificação.
git checkout <branch> Muda para uma ramificação específica.
git merge <branch> Mescla uma ramificação com a ramificação atual.
git pull Faz download do conteúdo de um repositório remoto e atualiza o repositório local.
git push Envia as alterações locais para o repositório remoto.
git remote Gerencia repositórios remotos, listando ou configurando repositórios conectados.
git remote add <nome> <url> Adiciona um repositório remoto com um nome específico.
git stash Armazena temporariamente mudanças não confirmadas para permitir mudar de branch sem perder o progresso.
git stash pop Aplica as mudanças armazenadas no stash de volta ao diretório de trabalho.
git reset <arquivo> Remove um arquivo da área de preparação, mantendo as mudanças no diretório de trabalho.
git reset --hard Redefine o repositório para o último commit, descartando todas as mudanças locais.
git rebase <branch> Reaplica commits em cima de outra branch para reorganizar o histórico.
git tag <nome> Marca um ponto específico no histórico do repositório, geralmente usado para releases.
